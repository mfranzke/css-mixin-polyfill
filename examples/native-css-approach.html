CSS Mixin/Macro Polyfill - Native vs Polyfill

The polyfill uses the same syntax as the upcoming native CSS specification

Same Syntax, Same Results

The CSS @mixin, @macro, and @apply specification is being developed by the CSS Working Group. This polyfill implements the same syntax so your code is forward-compatible with native browser support.

Native Browser Support (future)

Browsers will parse and apply @mixin/@macro/@apply natively.

  • Zero JavaScript overhead
  • Parsed by the CSS engine
  • Full cascade and specificity support
  • Hygienic variable renaming

Polyfill (today)

The polyfill processes the same syntax at runtime, transforming it into standard CSS.

  • Use the spec syntax today
  • Automatic stylesheet processing
  • Drop-in removal when browsers ship native support
  • PostCSS plugin for build-time transformation

1. @mixin with @result and @supports

The --gradient-text mixin from the CSS specification example, using @supports inside @result:

@mixin --gradient-text(--from <color>: mediumvioletred, --to <color>: teal, --angle: to bottom right) { --gradient: linear-gradient(var(--angle), var(--from), var(--to)); @result { color: var(--from); @supports (background-clip: text) or (-webkit-background-clip: text) { background: var(--gradient); color: transparent; -webkit-background-clip: text; background-clip: text; } } } h1 { @apply --gradient-text; } h2 { @apply --gradient-text(hotpink, dodgerblue, to right); }
Default gradient text (mediumvioletred → teal)
Custom gradient text (hotpink → dodgerblue)

2. @macro — No Parameters Needed

Macros are a simplified variant of mixins for reusable blocks that don't need arguments.

@macro --reset-list { margin: 0; padding: 0; list-style: none; } .clean-nav { @apply --reset-list; }

3. @mixin with @media inside @result

A responsive card mixin demonstrates how @media rules work inside @result, adapting to the viewport.

@mixin --responsive-card(--bg <color>: white, --shadow-intensity: 0.1) { @result { background: var(--bg); padding: 20px; box-shadow: 0 4px 15px rgba(0, 0, 0, var(--shadow-intensity)); @media (width >= 768px) { padding: 30px; } } } .demo-card { @apply --responsive-card; } .demo-card-accent { @apply --responsive-card(#f0f8ff, 0.2); }
Default responsive card — resize your browser to see padding change!
Responsive card with custom background and stronger shadow

4. @macro for Accessibility Patterns

@macro --visually-hidden { position: absolute; width: 1px; height: 1px; padding: 0; margin: -1px; overflow: hidden; clip: rect(0, 0, 0, 0); white-space: nowrap; border: 0; } .sr-only { @apply --visually-hidden; }

Migration Strategy

Step 1: Write your CSS using @mixin/@macro/@apply syntax today
Step 2: Include the polyfill script for current browser support
Step 3: Optionally use the PostCSS plugin for build-time transformation
Step 4: When browsers ship native support, remove the polyfill — your CSS stays the same
